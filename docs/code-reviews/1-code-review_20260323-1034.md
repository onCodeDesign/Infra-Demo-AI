### рџџў Suggestions

#### R3: Missing test for `"Customer {CustomerID}"` name fallback

- **File:** `Modules/Sales/Sales.Services.UnitTests/CustomerServiceTests.cs`
- **Commit:** `415764b6c6bb408f8c410aadab163a62c40c25c5`
- **Line(s):** (after L162)
- **Dimension:** Test Quality
- **Description:** The detailed design specifies a three-step name construction: (1) `CompanyName`, (2) `FirstName + LastName`, (3) `$"Customer {CustomerID}"`. The implementation correctly implements all three steps, but the test suite only validates steps 1 (`GetCustomersWithOverdueOrders_WithOverdueOrders_ReturnsCustomers`) and 2 (`GetCustomersWithOverdueOrders_HandlesNullCompanyName_UsesFirstLastName`). The third fallback вЂ” used when all name fields are null or whitespace вЂ” has no test coverage.
- **Suggested Fix:** Add a test `GetCustomersWithOverdueOrders_HandlesEmptyNameFields_UsesCustomerId` that creates a customer with `FirstName = ""`, `LastName = ""`, and `CompanyName = null`, and asserts the result `CustomerName` equals `"Customer 1"` (or the given CustomerID).

---

#### R4: `GetTarget` helper typed as concrete `FakeRepository` instead of `IRepository`

- **File:** `Modules/Sales/Sales.Services.UnitTests/CustomerServiceTests.cs`
- **Commit:** `415764b6c6bb408f8c410aadab163a62c40c25c5`
- **Line(s):** L166-L167
- **Dimension:** Test Quality
- **Description:** The `GetTarget` factory method accepts `FakeRepository repositoryStub` instead of `IRepository`. This unnecessarily couples the factory to the concrete test double. If a different test double (e.g., an `NSubstitute` substitute) were needed in a future test, a second overload would be required. Typing as the interface is the minimal, more flexible signature.
- **Suggested Fix:** Change the parameter type to `IRepository`: `private static CustomerService GetTarget(IRepository repositoryStub)`.

---

- **File:** `Modules/Sales/Sales.Services.UnitTests/CustomerServiceTests.cs`
- **Commit:** `415764b6c6bb408f8c410aadab163a62c40c25c5`
- **Line(s):** L197-L212
- **Dimension:** Test Quality
- **Description:** The project unit-testing convention names test doubles with a `Stub` or `Mock` suffix (e.g., `RepositoryStub`). The name `FakeRepository` uses a different prefix convention, creating inconsistency when future test classes in the same assembly follow the project standard.
- **Suggested Fix:** Rename `FakeRepository` to `RepositoryStub` and update all references in the file accordingly.
